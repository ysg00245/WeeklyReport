# AI 요약 — 모델, 비용, 프롬프트 설계

이 앱의 핵심 기능입니다. **프롬프트를 조직에 맞게 바꿀 수 있다는 점**이 강점이라,
어떻게 조립되고 어디를 고치면 되는지 정리합니다.

---

## 1. 무엇을 요약하나

한 팀의 그 주 주간보고를 모아서, **네 가지 관점** 중 하나로 재구성합니다.

| 유형 | `summary_type` | 결과물 |
|---|---|---|
| 구성원별 · 개인 | `member_individual` | 사람마다 한 블록 — 누가 뭘 했는지 |
| 구성원별 · 그룹 | `member_group` | 사람을 묶어 서술 — 인원 많을 때 |
| 프로젝트별 · 섹션 | `project` | 프로젝트마다 헤더 + 표 |
| 프로젝트별 · 단일표 | `project_flat` | 전체를 표 하나로 |
| 팀 전체 통합 | `team` | 팀 단위 서술 — 상위 보고에 적합 |
| 최종 취합 | `*_final` | 위 요약 + 팀장 본인 보고를 병합한 상부 보고서 |

같은 데이터로 관점만 바꿔 여러 번 뽑을 수 있고, 각각 따로 저장됩니다(`summaries` 테이블).

## 2. 추천 모델

기본값은 **`claude-sonnet-4-6`** 입니다 (`routers/ai.py`).

| 모델 | 적합도 | 비고 |
|---|---|---|
| Sonnet 계열 | ⭐ **권장** | 표 구조 유지·한국어 존댓말·요약 압축의 균형이 좋음 |
| Haiku 계열 | 비용 우선일 때 | 훨씬 저렴하고 빠르지만, 복잡한 표 형식에서 구조가 흐트러지는 경우가 있음 |
| Opus 계열 | 품질 우선일 때 | 비용 대비 체감 개선이 크지 않았음 |

**모델을 바꾸려면** `routers/ai.py` 의 `_sync_call_claude()` 안 `model=` 값을 수정하세요.
최신 모델명은 [Anthropic 문서](https://docs.anthropic.com/en/docs/about-claude/models)에서 확인할 수 있습니다.

> ⚠️ 이 저장소를 운영 중인 조직에서는 모델명을 임의로 바꾸지 마세요.
> 출력 형식이 프롬프트에 강하게 맞춰져 있어 모델을 바꾸면 표 구조가 달라질 수 있습니다.
> 바꿀 때는 실제 데이터로 각 요약 유형을 한 번씩 뽑아보고 결정하세요.

### 출력 길이 상한

`max_tokens=4000` 입니다. 예전에 2000이었을 때 프로젝트별 표 요약이 중간에 잘려
**표가 깨지고 뒷 섹션이 통째로 사라지는** 문제가 있었습니다.

출력 토큰은 실제 생성된 만큼만 과금되므로, 상한을 올려도 평소 비용은 늘지 않습니다.
상한에 도달하면 잘린 결과를 저장하지 않고 명시적으로 실패시킵니다.

## 3. 예상 비용

### 요청 1회의 토큰 규모

```
입력 = 시스템 프롬프트(페르소나 + 출력 형식 + 팀 추가 지침) + 팀원 보고 본문
     ≈ 2,000 ~ 6,000 토큰   (형식 프롬프트가 상세할수록 커짐)

출력 ≈ 800 ~ 2,500 토큰     (요약 유형과 인원수에 따라)
```

프롬프트를 커스터마이즈하면 입력이 커집니다. 실제 운영 사례에서 프로젝트별 출력 형식
프롬프트가 14,000자(약 7,000토큰)까지 늘어난 적이 있으니, 상세한 형식을 넣을수록
입력 비용이 지배적이 됩니다.

### 계산 방법

정확한 단가는 모델·시점마다 다르므로 [Anthropic 요금 페이지](https://www.anthropic.com/pricing)를
확인하세요. 계산식은 이렇습니다.

```
1회 비용 = (입력토큰 ÷ 1,000,000 × 입력단가) + (출력토큰 ÷ 1,000,000 × 출력단가)
월 비용  = 1회 비용 × 팀수 × 주당 생성횟수 × 4.3주
```

### 실사용 감각

주간보고는 **일주일에 한 번, 팀당 몇 번** 생성하는 패턴입니다. 이 앱을 쓰는 조직의
실제 사용량은 이렇습니다.

```
10개 팀 × 주 3회 생성(관점 바꿔가며) × 4.3주 ≈ 월 130회
```

같은 시간대에 수백 명이 동시에 호출하는 서비스가 아니라, **월 수백 회 규모**입니다.
LLM 서비스 중에서는 비용 부담이 작은 축에 속합니다.

### 비용을 줄이려면

1. **출력 형식 프롬프트를 간결하게** — 입력 토큰이 매번 곱해집니다. 가장 효과가 큽니다
2. **자주 쓰는 관점만 사용** — 4가지를 매번 다 뽑을 필요는 없습니다
3. **저장본 재사용** — 생성 결과는 DB에 저장됩니다. 같은 주차를 다시 열 때 재호출하지 않습니다
4. **Haiku 계열 검토** — 형식이 단순한 조직이라면 충분할 수 있습니다

`ANTHROPIC_API_KEY` 를 비워두면 AI 기능만 비활성화되고 나머지는 정상 동작합니다.
비용이 부담되면 요약 없이 쓰다가 나중에 켜도 됩니다.

## 4. 프롬프트 구조 — 이 앱의 강점

프롬프트가 **하나의 통짜 문자열이 아니라 조각으로 나뉘어 조립**됩니다.
덕분에 조직 특성만 따로 덧붙일 수 있습니다.

### 조립 순서

```
[페르소나]                    ai_persona
      +
[팀별 추가 지침 — 페르소나]     ai_team_extra_persona      ← 선택
      +
[출력 형식]                   ai_format_<유형>
      +
[팀별 추가 지침 — 출력 형식]    ai_team_extra_<유형>        ← 선택
      ↓
  시스템 프롬프트 완성 → 팀원 보고 본문과 함께 호출
```

### 프롬프트 키

**기본 프롬프트** — 요약의 뼈대

| 키 | 역할 |
|---|---|
| `ai_persona` | 어투·관점·원칙 (모든 요약 공통) |
| `ai_format_member_individual` | 구성원별 개인 출력 형식 |
| `ai_format_member_group` | 구성원별 그룹 출력 형식 |
| `ai_format_project` | 프로젝트별 섹션형 |
| `ai_format_project_flat` | 프로젝트별 단일표 |
| `ai_format_team` | 팀 전체 통합 |
| `ai_format_final` | 최종 취합(상부 보고서) |
| `ai_assist_prompt` | 작성 도우미(구성원 보고 초안 첨삭) |

**추가 지침** — 기본 프롬프트를 지우지 않고 덧붙이는 용도

| 키 | 붙는 위치 |
|---|---|
| `ai_team_extra_persona` | 페르소나 뒤 — 모든 요약에 공통 적용 |
| `ai_team_extra_member` | 구성원별 요약일 때만 |
| `ai_team_extra_project` | 프로젝트별 요약일 때만 |
| `ai_team_extra_team` | 팀 전체 요약일 때만 |

추가 지침은 짧게 쓰는 것이 좋습니다. 예시:

```
- 각 구성원 라인 끝에 이번 주 핵심 이슈를 1줄 덧붙일 것
- 임원 보고용 어투. 결론 → 리스크 → 요청사항 순서로 구획할 것
- 프로젝트별 일정 마일스톤과 위험도를 함께 표기할 것
```

### 어디서 수정하나

| 대상 | 위치 | 권한 |
|---|---|---|
| 기본 프롬프트 7종 | 시스템 관리자 콘솔 → **프롬프트** 탭 | 시스템 관리자 |
| 팀별 추가 지침 4종 | 관리자 콘솔 → **양식 설정** 탭 | 팀 관리자 |

**코드의 `DEFAULT_*` 상수는 DB에 값이 없을 때만 쓰이는 초기값입니다.**
이미 운영 중인 배포는 DB에 값이 있으므로, 코드만 고치면 반영되지 않습니다.

## 5. ⚠️ 알려진 제약 — 프롬프트 적용 범위

**현재 구현은 프롬프트를 팀 단위로만 적용합니다.**

`_load_prompt()` 가 요청한 팀의 `team_id` 로만 조회하고, 없으면 코드 기본값으로 떨어집니다.
전역(`team_id = 0`)으로 저장된 값을 찾아보는 단계가 없습니다.

그래서 실제 동작은 이렇습니다.

```
프롬프트를 커스터마이즈한 팀   → 그 팀의 값 사용          ✅
아무것도 설정하지 않은 팀      → 코드의 DEFAULT_* 사용     ✅
시스템 관리자가 저장한 전역값  → 어느 팀에도 적용되지 않음  ⚠️
```

**결과**: 여러 팀에 같은 프롬프트를 쓰려면 팀마다 각각 저장해야 합니다.
"전사 공통 프롬프트를 한 번에 바꾸기"는 아직 동작하지 않습니다.

### 고치려면

`_load_prompt()` 에 전역 폴백 한 단계를 넣으면 됩니다.

```python
# 팀 값 → 없으면 전역(team_id=0) → 없으면 코드 기본값
```

이렇게 하면 의도했던 3계층이 완성됩니다:
**코드 기본값 < 전사 공통 < 팀별 커스터마이즈**

⚠️ 다만 이미 운영 중인 배포라면 주의하세요. 그동안 무시되던 전역 설정값이 갑자기
활성화되면서 요약 결과가 바뀔 수 있습니다. 고치기 전에 `settings` 테이블에
`team_id = 0` 인 `ai_*` 행이 있는지 먼저 확인하세요.

## 6. 프롬프트를 고칠 때의 요령

- **출력 형식은 예시를 보여주는 게 가장 확실합니다.** "표로 정리하세요"보다
  실제 표 예시를 넣으면 형식이 훨씬 안정적입니다
- **한 번에 하나씩 바꾸세요.** 페르소나와 형식을 동시에 고치면 뭐가 효과였는지 모릅니다
- **실제 데이터로 확인하세요.** 인원이 적은 주와 많은 주는 결과가 다릅니다
- **표가 깨지면 출력 길이를 의심하세요.** `max_tokens` 상한에 걸리면 명시적으로 실패하지만,
  형식 프롬프트가 너무 장황해서 모델이 헤매는 경우도 있습니다
- 기본 프롬프트를 통째로 덮기보다 **추가 지침으로 덧붙이는 쪽**이 유지보수가 쉽습니다
