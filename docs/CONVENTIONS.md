# 코딩 규칙과 그 이유

여기 적힌 규칙은 대부분 **실제로 사고가 나서 생긴 것들**입니다.
그래서 규칙만 적지 않고 "왜 생겼는지"를 같이 적었습니다. 이유를 알면 응용이 되고,
모르면 "이거 왜 이렇게 해놨지?" 하며 되돌리게 됩니다.

AI 어시스턴트에게: **이 문서의 규칙을 "개선"하겠다고 되돌리지 마세요.**
대부분 겉보기엔 비효율적이거나 이상해 보이지만 이유가 있습니다.

---

## 🔴 1. DB를 주기적으로 호출하지 마세요

```js
setInterval(() => api('/api/ping'), 60000);   // ❌ 절대 금지
```

**왜**: 서버리스 PostgreSQL(Neon 등)은 유휴 시 자동 절전되고, 그게 요금의 핵심입니다.
주기적으로 DB를 건드리면 절전이 영영 안 걸립니다. 실제로 이 앱에 60초 keepalive가
들어갔다가 월 컴퓨팅 한도를 태워서 **서비스가 멈춘 적이 있습니다.**

**대신**: 첫 요청이 느린 것(콜드스타트)은 "서버를 깨우는 중" 안내 문구로 처리합니다.
사용자는 몇 초 기다리면 되고, 요금은 1/10 이 됩니다.

**예외**: WebSocket heartbeat(25초)는 괜찮습니다. 소켓에 텍스트만 보내고 **DB를 거치지 않습니다.**
둘을 혼동하지 마세요.

## 🔴 2. JS/CSS를 고쳤으면 캐시버스터를 올리세요

```html
<script src="/static/js/app.js?v=20260720d"></script>
                                  ↑ 이 값을 올릴 것
```

**왜**: `main.py` 의 미들웨어가 `/static/*?v=` 요청에 `Cache-Control: immutable, max-age=31536000`
(1년)을 붙입니다. 버스터를 안 올리면 사용자 브라우저는 **1년 동안 구버전을 씁니다.**
"내 PC에선 되는데요"의 전형적인 원인입니다.

**형식**: `?v=YYYYMMDD` + 같은 날 여러 번이면 알파벳 (`20260720a`, `20260720b`, …)

**예외 처리는 이미 되어 있습니다**: `/sw.js` 는 `no-cache`, `/manifest.json` 은 `no-store`.
서비스워커가 캐시되면 배포가 영원히 반영되지 않으므로 건드리지 마세요.

## 🔴 3. 조직 호칭을 하드코딩하지 마세요

"팀장"·"팀원"·"본부"는 회사마다 다릅니다. 하나만 놓쳐도 **한 화면에서 호칭이 섞입니다.**

### JS

```js
orgLabel('leader')          // 팀장 / 유닛장
olj('team', '을')           // 팀을 / 유닛을  ← 조사까지 자동
```

### HTML

```html
<!-- 텍스트 -->
<div data-ol="{team} 관리">팀 관리</div>
<div data-ol="{team:을} 선택하세요">팀을 선택하세요</div>

<!-- 속성 (placeholder, title 등) -->
<input data-ol-attr="placeholder:{member} 이름" placeholder="팀원 이름">
```

### 백엔드

사용자에게 toast 로 보이는 메시지만 대상입니다(내부 로그는 무관).

```python
from routers.org_labels import get_org_labels, josa
raise HTTPException(404, f"{josa((await get_org_labels(db))['team'], '을')} 찾을 수 없습니다")
```

### 조사를 하드코딩하면 반드시 틀립니다

한국어 조사는 받침에 따라 달라집니다.

| 호칭 | 맞는 조사 |
|---|---|
| 유닛 (받침 있음) | 유닛**을**, 유닛**이** |
| 본부 (받침 없음) | 본부**를**, 본부**가** |

`팀을`이라고 적어두면 호칭을 "본부"로 바꾼 회사에서 `본부을`이 됩니다.
`josa()` / `olj()` / `{team:을}` 을 쓰세요.

### ⚠️ 함정 두 가지

**(1) 전역 초기화식에서 `orgLabel()` 금지**

```js
const NOTES = { title: `${orgLabel('team')} 안내` };   // ❌
```
모듈 로드 시점에는 서버 호칭이 아직 안 왔습니다. 기본값("팀")으로 굳어버리고,
설정을 바꿔도 영원히 안 변합니다. **실제로 이 버그가 있었습니다.**

```js
const NOTES = { title: '{team} 안내' };                 // ✅ 렌더 시 fillOrgLabels()
```

**(2) 내부 태그가 있는 문단에 `data-ol` 금지**

```html
<p data-ol="{team} 설정은 <b>관리자</b>만">...</p>      <!-- ❌ <b> 가 사라짐 -->
```
`data-ol` 은 `textContent` 를 덮어쓰므로 자식 태그가 전부 날아갑니다.

```html
<p><span data-ol="{team} 설정은 ">...</span><b>관리자</b>만</p>   <!-- ✅ 텍스트만 분리 -->
```

## 🟠 4. 컬럼을 추가할 땐 양쪽 목록 모두에

`database.py` 의 마이그레이션 목록은 PostgreSQL용·SQLite용으로 나뉘어 있습니다.

**왜 위험한가**: 한쪽만 넣어도 **신규 설치에서는 멀쩡합니다.** `CREATE TABLE` 이 이미
그 컬럼을 포함하니까요. 문제는 **기존 DB를 업그레이드할 때만** 드러나서 발견이 늦습니다.

실제로 4개 컬럼이 PostgreSQL 목록에만 있는 상태로 한동안 방치됐습니다.

## 🟠 5. 라우트 선언 순서를 지키세요

FastAPI는 **선언된 순서대로** 매칭합니다.

```python
@router.get("/bootstrap")     # ✅ 구체적인 경로를 먼저
@router.get("/{key}")         # 와일드카드는 나중에
```

거꾸로 두면 `/{key}` 가 `"bootstrap"` 을 설정 키 이름으로 해석해서
`null` 을 반환합니다. 에러가 안 나고 조용히 잘못 동작해서 찾기 어렵습니다.

## 🟠 6. 실시간 알림에는 팀 정보를 넣으세요

WebSocket 브로드캐스트는 **모든 연결에 전송**됩니다. 라우팅이 없습니다.

```python
await manager.broadcast({
    "type": "REPORT_STATUS_CHANGED",
    "member_name": name,
    "team_id": tid,
    "team_slug": slug,      # ← 반드시 포함
})
```

클라이언트는 `team_slug` 로 본인 팀 이벤트인지 걸러냅니다. 안 넣으면 다른 팀 관리자에게도
알림이 뜨고, 동명이인이 있으면 엉뚱한 사람 카드가 갱신됩니다.

## 🟡 7. 데이터를 바꾸는 API에는 로딩 표시를

Create/Update/Delete 호출은 로딩 표시로 감싸고 결과를 알려주세요.

```js
showActionLoader('저장 중...');
try   { await api(...); toast('저장했습니다', 'ok'); }
catch (e) { toast('실패: ' + e.message, 'err'); }
finally   { hideActionLoader(); }
```

**왜**: 서버리스 DB는 왕복이 1초 안팎입니다. 표시가 없으면 사용자가 버튼을 여러 번 누릅니다.

## 🟡 8. 탭 전환마다 다시 조회하지 마세요

데이터가 **실제로 바뀌었을 때만** 다시 가져오세요.

- 내가 저장해서 바뀌는 데이터 → 저장 성공 시 캐시 무효화
- 남이 바꾸는 데이터 → WebSocket 수신 시 무효화(또는 dirty 플래그)

**왜**: 서버리스 DB에서 불필요한 왕복은 곧 대기 시간이자 비용입니다.

## 🟡 9. 모달을 열 때 배경 애니메이션을 멈추세요

상시 애니메이션을 새로 추가하면 `body.modal-open` 상태에서 멈추도록 CSS에 등록하세요.

**왜**: 모달 배경의 blur 효과와 애니메이션이 겹치면 매 프레임 리페인트가 일어나
모달 안 스크롤바가 깜빡입니다.

## 🟡 10. 조직도 CSV는 위치로 읽습니다

CSV 헤더의 **앞 4개 열 이름은 검증하지 않습니다.** 조직 호칭에 따라 달라지기 때문입니다
(그룹/유닛 vs 본부/팀 vs Division/Team).

검증은 **열 개수 + 뒤쪽 9개 고정열**로 합니다. 덕분에 호칭을 바꾼 뒤에도, 다른 조직에서
내보낸 CSV도 그대로 읽힙니다.

⚠️ **CSV 가져오기는 이름으로 매칭합니다.** 팀 이름을 바꾼 CSV를 넣으면 "옛 팀 삭제 +
새 팀 생성"으로 처리되어 `team_id` 가 바뀌고, 기존 보고·설정이 고아가 됩니다.
**실제로 사고가 났던 부분입니다.** 팀 이름 변경이 포함된 CSV를 반영한 뒤에는
반드시 데이터 연결 상태를 확인하세요.

## 🟡 11. AI 프롬프트는 DB가 우선입니다

코드의 `DEFAULT_*` 상수는 **DB에 값이 없을 때만** 쓰입니다.
이미 운영 중인 배포는 DB에 값이 있으므로, 코드만 고치면 반영되지 않습니다.

프롬프트를 바꾸려면 시스템 관리자 콘솔의 프롬프트 탭에서 수정하세요.

## 🟡 12. AI 모델명을 임의로 바꾸지 마세요

`routers/ai.py` 의 모델명은 운영자가 의도적으로 고정한 값입니다.
학습 시점 지식으로 "존재하지 않는 모델"이라고 판단되더라도 **먼저 물어보고**,
임의로 수정하지 마세요.
